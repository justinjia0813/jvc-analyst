# /ic-memo 投决备忘录

`/ic-memo` 用于把已完成的项目材料合成为投决会备忘录初稿。它是整理材料的工具，不替用户决策。

## 输入

用户可提供以下材料：

- deck、BP、财务表、交易条款、历史融资信息
- `/prescreen` 初筛纪要
- 尽调笔记、客户/专家访谈纪要、背调记录
- `/bear-case` 反向论证
- `/track-research` 赛道研究和可比公司表
- 用户给出的核心投资逻辑或投委会关注点

如果关键输入缺失，不要补全空白；用 `[需要用户提供]` 或 `[未核实]` 标出来，并放入 `待决事项`。

## 工作流

1. 列出已读取材料和来源范围。
2. 抽取所有数字、交易条款和关键事实，逐条保留来源标签。
3. 区分三类内容：
   - `事实`：来自素材、访谈或公开来源
   - `判断`：基于事实形成的分析
   - `待验证`：缺来源、缺交叉验证或只有单点信源
4. 按 `templates/ic-memo-template.md` 合成 memo 初稿。
5. 将用户给出的核心投资逻辑展开为“可讨论的论点”，不要改写成最终决策。
6. `风险与反方观点` 必须直接吸收 `/bear-case` 或现有反面材料，篇幅不得短于 `投资逻辑`。
7. `估值与条款` 只摆事实和参照，不下“贵/便宜/应该投”的结论。
8. 末尾列出 `待决事项`、`未覆盖的关键问题` 和 `[未核实]` 清单。

## 输出格式

使用 `templates/ic-memo-template.md` 作为固定结构。

核心段落必须包含：

- `交易摘要`
- `公司概况`
- `市场与竞争`
- `产品与技术`
- `团队`
- `财务与单位经济`
- `投资逻辑`
- `风险与反方观点`
- `估值与条款`
- `待决事项`
- `未覆盖的关键问题`
- `来源索引`

## 质量红线

- 不替用户决策，不写“建议投资”“不建议投资”“pass 掉”等终局判断。
- 不输出风险评级、推荐评级、投票意见，除非用户明确要求并在输出前确认边界。
- 所有市场规模、估值、融资金额、收入、毛利、用户数、增长率都必须有来源。
- 创始人未经验证的说法标 `[创始人自述]`。
- 单一信源的负面信息标 `[待交叉验证]`，不能直接作为结论。
- 删除没有证据的修饰词，例如“领先”“颠覆性”“护城河明显”。
