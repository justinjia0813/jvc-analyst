# JVC Knowledge Tree Builder

`jvc-knowledge-tree-builder` reads a local VC track, project, Obsidian, or source folder and turns existing files into a recursive, source-backed knowledge model.

## Outputs

- `knowledge_tree.md`: human-readable recursive question tree.
- `knowledge_graph.mmd`: Mermaid graph for tree and cross-links.
- `nodes.json`: structured nodes and edges for future UI consumption.
- `evidence_index.md`: source path index and node-to-source mapping.
- `open_questions.md`: unresolved questions grouped by branch.
- `source_manifest.json`: optional deterministic source inventory generated by `scripts/collect_sources.py`.

## Basic Use

From the skill directory:

```bash
python3 scripts/collect_sources.py /path/to/topic-folder --output /path/to/output/source_manifest.json
```

Then use the manifest and readable source files to produce the five output artifacts following `references/output-contract.md`.

## Boundary

Use this after source material exists. For first-pass sector research, use `/jvc-track-research`; for company comparison, market sizing, bull case, bear case, or IC memo, use the matching `jvc-*` skill.

## Validation

Run:

```bash
python3 scripts/collect_sources.py . --max-files 20 --output /tmp/jvc-knowledge-tree-builder-smoke.json
python3 scripts/check_package.py
```
