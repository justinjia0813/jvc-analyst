# /jvc-track-research — 产业知识图谱

给细分赛道名，快速建立认知框架。

## 输入

- 细分赛道名称 / 关键词
- 可选：已知玩家、技术路线、关注环节或地域范围

## 执行步骤

### 1. 联网搜索

使用 web search 搜集公开信息（论文/专利、监管政策、公司官网、行业报告、新闻、上市公司公告）。先做术语和缩写消歧，再检索中英文同义词。按可信度分层使用来源：论文/监管/公告 > 公司官网 > 主流媒体 > 券商/二级市场材料 > 自媒体。

### 2. 结构化输出

按以下骨架组织（无信息的章节写"暂无公开信息"，不编造）：

**A. 行业定义与边界**
- 这个赛道是什么、不是什么
- 上下游边界在哪

**B. 行业简史**
- 从技术诞生或产业萌芽写起，而不是只列最近事件
- 说明每一阶段为什么出现：核心瓶颈、关键突破、资本/政策/需求变化
- 用连续脉络呈现底层逻辑：上一阶段解决了什么，又暴露了什么新问题
- 最后判断当前处于科研验证、工程样机、示范应用、商业放量中的哪一段

**C. 技术路线**
- 先讲原理和设计差异，再列代表公司
- 每条路线说明：物理/工程原理、核心设计选择、关键部件、优势、约束、成熟度
- 比较路线时聚焦 trade-off：性能、成本、可靠性、制造难度、监管/安全、商业化时间
- 代表公司只作为路线样本，不能用公司报道替代技术解释

**D. 产业链图谱**
- 上游 → 中游 → 下游，各环节的核心玩家和价值分配

**E. 产业趋势**
- 分成政策趋势、技术趋势、市场趋势三类
- 不只列单点事件，要解释事件背后的连续性和驱动力
- 政策趋势：监管口径、补贴/基金、国产替代、地方产业集群、合规门槛
- 技术趋势：核心瓶颈、正在被什么方法突破、仍未解决的工程问题
- 市场趋势：谁会先付费、短中长期应用顺序、采购/验证周期、商业化前置市场
- 给出 3-5 年展望，并标出哪些是已发生事实、哪些是推断

**F. 关键玩家分层**
- 按上市公司 / 成长期公司 / 早期公司分层
- 每个玩家一句话定位

**G. 监管 / 政策 / 标准**
- 现行政策
- 在讨论中的政策变化

**H. 投资相关问题**
- 进入这个赛道需要回答的核心问题
- 后续可进入 `/jvc-comps-dd`、`/jvc-market-sizing`、`/jvc-bull-case`、`/jvc-bear-case` 的线索

## 输出格式

单一 Markdown 文档，按上述 A-H 结构。

## 硬约束

- 所有数据点标注来源 `[来源: xxx]`
- 找不到来源标 `[未核实]`
- 不编造市场规模数字、玩家数据、政策条文
- 不写"这是一个万亿级市场"之类无来源的判断
- 不把技术里程碑混写：实验点亮、样机验证、稳定运行、客户试用、商业收入、规模交付必须分清
- 不用公司 PR 或融资报道替代技术路线解释
- 中文输出，术语首次出现附中文解释
