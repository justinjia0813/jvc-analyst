# /jvc-track-research — 产业知识图谱

给细分赛道名，生成第一轮投资研究地图。它解决“这个赛道到底怎么运转、哪里值得继续查”，不是直接给投资结论。

## 输入

- 细分赛道名称 / 关键词
- 可选：地域范围、阶段偏好、已知玩家、技术路线、关注环节、时间范围

如果用户没给地域和阶段，默认按“全球及中国市场、早期 VC、未来 3-5 年”处理，并在开头写明假设。

VC = Venture Capital，风险投资 / 创业投资；这里指围绕创业公司融资与投资判断的研究语境。

## 执行步骤

### 1. 先定研究口径

先输出一个很短的研究设定：

- 赛道定义：研究对象是什么
- 排除边界：哪些相邻赛道不纳入
- 地域 / 时间 / 投资阶段假设
- 关键同义词、英文词、缩写消歧
- 本轮研究要支持的下一步判断，以及明确不回答的问题

所有英文缩写首次出现时必须给出：英文全称、中文全称、简要解释。

### 2. 先判断发展阶段，再分配研究权重

不要把所有行业套进同一张静态模板。先提出产业生命周期假设，并说明证据和置信度：

- 导入期：重点验证商业模式的**可行性**，即需求是否真实、谁付费、产品能否交付并留下正向单位经济
- 成长期：重点验证市场的**规模性**，即增长来自渗透、替代还是扩品类，供给能否跟上
- 成熟期：重点验证**防守性**与**盈利性**，即替代威胁、护城河、横向集中度和纵向议价权
- 衰退或再生期：重点验证替代速度、供需收缩差、第二增长曲线是否成立

渗透率、销量增速、客户验证阶段和产能利用率可以作为阶段证据，但不能把任何单一阈值当作跨行业通用分界线。阶段证据不足时写“待判定”，同时保留两种最可能解释。

基于上述判断列出 3-7 条可证伪假设。每条写清：当前判断、支持证据、反证条件、下一步验证动作。研究优先级由“对下一步判断的重要性 × 当前证据缺口”决定，不由资料是否容易找到决定。

### 3. 假设驱动地联网搜索

使用 web search 搜集公开信息。覆盖以下来源类型中的全类；确实找不到时说明缺口：

- 论文 / 专利 / 标准
- 监管政策 / 政府文件 / 行业协会
- 公司官网 / 产品文档 / 招股书 / 年报
- 行业报告 / 券商或咨询报告
- 主流媒体 / 融资新闻
- 客户、渠道、开发者或采购侧材料

先做术语和缩写消歧，再检索中英文同义词。按可信度分层使用来源：论文/监管/公告 > 公司官网/招股书/年报 > 行业报告 > 主流媒体 > 券商/二级市场材料 > 公司 PR/融资新闻 > 自媒体。

PR = Public Relations，公共关系 / 公关稿；只能作为公司自述线索，不能当作技术或收入事实。

搜索按“初始假设 → 拆成互不重叠的问题 → 为每个问题寻找验证指标”推进。不要先堆资料再寻找观点。主动搜索与初始判断相反的证据，并记录来源的定义、统计口径、立场和日期。

### 4. 建立因果链和周期位置

把长期趋势与短期周期分开。至少检查以下周期，确实不适用时说明原因：

- 需求 / 技术采纳周期：客户从认知、试用、采购到复购处在哪一段
- 产能 / 库存周期：供不应求、扩产、过剩出清或格局改善
- 竞争 / 盈利周期：进入者、价格、毛利和集中度如何变化
- 资本 / 融资周期：融资可得性、估值预期、扩张意愿和风险偏好如何变化
- 政策 / 技术冲击：外部因素如何改变上述周期，而不是把单点事件当成趋势

对每个适用周期给出“上行早期 / 扩张中段 / 过热风险 / 下行消化 / 筑底早期 / 证据不足”之一，并列领先指标、滞后指标、反向信号和置信度。只判断位置区间和概率倾向，不预测精确拐点或时间。

用因果链解释“为什么”：当前状态 → 参与者行为 → 供需、竞争或融资变化 → 下一阶段压力。特别检查两类反身性：好景气是否诱发过度扩产、宽松融资和低门槛进入；坏景气是否迫使出清、收缩供给并改善未来格局。不同周期冲突时保留冲突，不强行合成一个方向。

### 5. 交叉验证与证据分层

对关键结论做最小交叉验证：

- 市场空间、增速、渗透率、价格、出货量：至少两类来源互证；不能互证则标 `[未核实]`
- 技术里程碑：区分论文/实验、样机、试点、客户试用、收入、规模交付
- 玩家状态：区分上市公司、成熟公司、成长期公司、早期公司、实验室项目
- 政策：区分已生效文件、征求意见、地方扶持、产业口号
- 周期位置：至少用两个不同维度的指标；只有叙事没有指标时标 `[低置信度]`

来源冲突时不取看似精确的平均数。先检查定义、地域、时间、样本和测算方法，再给区间或并列口径并解释差异。

### 6. 从资料提炼判断

重要判断按“证据 → 机制 → 对投资研究的含义 → 反证条件”表达。事实、推断和未知分别标记；连续追问“这意味着什么”，直到结论能改变后续验证顺序或尽调重点。不能改变任何下一步动作的信息，只作为背景，不进入一页快照。

### 7. 停止搜集

高优先级假设已经被两类来源支持、被反证，或已明确写出无法补齐的关键证据时，停止扩展。连续两轮搜索没有改变生命周期判断、核心因果链或前三项验证优先级时，也停止搜集。若定义、阶段或关键数据仍缺失，缩小结论并交付证据缺口，不用低质量来源填满章节。

### 8. 结构化输出

按以下骨架组织。无信息的章节写“暂无公开信息”，不编造。

**0. 研究设定与一页快照**
- 研究口径、排除边界、默认假设
- 当前产业生命周期、判断证据、置信度和本轮研究优先级
- 可行性 / 规模性 / 防守性 / 盈利性四项初判
- 3-5 条结论先行的关键判断，每条带证据、机制和含义
- 3-5 条最重要的不确定性与反证条件
- 适用周期的位置矩阵，以及一条把关键周期串起来的核心因果链
- 后续最该进入哪个 skill：`/jvc-knowledge-tree-builder`、`/jvc-comps-dd`、`/jvc-market-sizing`、`/jvc-bull-case`、`/jvc-bear-case`

**A. 行业定义与边界**
- 这个赛道是什么、不是什么
- 上下游边界在哪

**B. 行业简史与产业生命周期**
- 从技术诞生或产业萌芽写起，而不是只列最近事件
- 说明每一阶段为什么出现：核心瓶颈、关键突破、资本/政策/需求变化
- 用连续脉络呈现底层逻辑：上一阶段解决了什么，又暴露了什么新问题
- 最后判断当前处于导入、成长、成熟、衰退或再生中的哪一段，并说明细分市场分化

**C. 技术路线与商业可行性**
- 先讲原理和设计差异，再列代表公司
- 每条路线说明：物理/工程原理、核心设计选择、关键部件、优势、约束、成熟度
- 比较路线时聚焦 trade-off：性能、成本、可靠性、制造难度、监管/安全、商业化时间
- 对每条主路线补充：目标客户、付费理由、交付前提、单位经济是否有证据成立
- 代表公司只作为路线样本，不能用公司报道替代技术解释

**D. 产业链图谱**
- 上游 → 中游 → 下游，各环节的核心玩家和价值分配
- 标出谁掌握议价权、谁承担交付成本、谁最接近客户预算

**E. 产业趋势、景气度与周期位置**
- 分成政策趋势、技术趋势、市场趋势三类
- 不只列单点事件，要解释事件背后的连续性和驱动力
- 政策趋势：监管口径、补贴/基金、国产替代、地方产业集群、合规门槛
- 技术趋势：核心瓶颈、正在被什么方法突破、仍未解决的工程问题
- 市场趋势：谁会先付费、短中长期应用顺序、采购/验证周期、商业化前置市场
- 给出 3-5 年展望，并标出哪些是已发生事实、哪些是推断
- 用需求、产能、竞争、资本四类周期指标定位当前位置，区分领先与滞后指标
- 列出会让当前位置判断失效的反向信号，不预测精确拐点

**F. 关键玩家分层**
- 按上市公司 / 成长期公司 / 早期公司分层
- 每个玩家一句话定位
- 不把融资额等同于技术领先，不把媒体曝光等同于商业化

**G. 监管 / 政策 / 标准**
- 现行政策
- 在讨论中的政策变化

**H. 投资相关问题**
- 用“假设 / 证据 / 机制 / 反证条件 / 下一步动作”维护问题账本
- 分成：客户付费、技术壁垒、交付复制、单位经济、竞争格局、监管风险

**I. 后续工作交接包**
- 给 `/jvc-knowledge-tree-builder`：建议沉淀的资料文件夹结构
- 给 `/jvc-comps-dd`：候选竞品 / 可比公司清单和比较维度
- 给 `/jvc-market-sizing`：可建模变量、口径、需要补的数据
- 给 `/jvc-bull-case`：最强 3 条正向假设
- 给 `/jvc-bear-case`：最危险 3 条反向假设
- 给 `/jvc-meeting-notes` / `/jvc-talk-notes`：建议访谈对象和问题清单

## 输出格式

单一 Markdown 文档，按上述 0 和 A-I 结构。优先使用表格承载事实、来源、判断和待验证项；短解释可以用段落。

## 硬约束

- 所有数据点标注来源 `[S1]`、`[S2]`
- 文末必须有“来源索引”，列出来源标题、机构/作者、日期、链接、来源类型、可信度
- 找不到来源标 `[未核实]`
- 不编造市场规模数字、玩家数据、政策条文
- 不写"这是一个万亿级市场"之类无来源的判断
- 不把技术里程碑混写：实验点亮、样机验证、稳定运行、客户试用、商业收入、规模交付必须分清
- 不用公司 PR 或融资报道替代技术路线解释
- 不把时间顺序当作因果关系，不把单一指标或近期趋势线性外推为未来
- 不混淆长期增长趋势、产业生命周期和短期景气周期
- 不声称能预测精确顶部、底部、拐点或发生时间；周期判断必须写置信度和反向信号
- 不输出“建议投资 / 不建议投资”；只输出研究问题、证据缺口和后续尽调方向
- 中文输出，术语首次出现附中文解释
