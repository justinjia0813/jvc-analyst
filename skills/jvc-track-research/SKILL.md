# /jvc-track-research — 产业知识图谱

给细分赛道名，快速建立认知框架。

## 输入

- 细分赛道名称 / 关键词
- 可选：已知玩家、技术路线、关注环节或地域范围

## 执行步骤

### 1. 联网搜索

使用 web search 搜集公开信息（行业报告摘要、新闻、政策文件、上市公司公告），优先中文信息源（36kr、亿欧、艾瑞、甲子光年、IT 桔子、产业信息网等），辅以英文源。

### 2. 结构化输出

按以下骨架组织（无信息的章节写"暂无公开信息"，不编造）：

**A. 行业定义与边界**
- 这个赛道是什么、不是什么
- 上下游边界在哪

**B. 行业简史**
- 关键里程碑时间线（技术突破、政策转折、标志性事件）

**C. 技术路线**
- 主流技术路线及其优劣对比
- 正在发生的技术路线迁移

**D. 产业链图谱**
- 上游 → 中游 → 下游，各环节的核心玩家和价值分配

**E. 产业趋势**
- 增长驱动力（需求/政策/技术）
- 3-5 年展望

**F. 关键玩家分层**
- 按上市公司 / 成长期公司 / 早期公司分层
- 每个玩家一句话定位

**G. 监管 / 政策 / 标准**
- 现行政策
- 在讨论中的政策变化

**H. 投资相关问题**
- 进入这个赛道需要回答的核心问题
- 后续可进入 `/jvc-comps-dd`、`/jvc-market-sizing`、`/jvc-bull-case`、`/jvc-bear-case` 的线索

## 输出格式

单一 Markdown 文档，按上述 A-H 结构。

## 硬约束

- 所有数据点标注来源 `[来源: xxx]`
- 找不到来源标 `[未核实]`
- 不编造市场规模数字、玩家数据、政策条文
- 不写"这是一个万亿级市场"之类无来源的判断
- 中文输出，术语首次出现附中文解释
