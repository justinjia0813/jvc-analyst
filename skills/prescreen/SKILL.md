# /prescreen 初筛

`/prescreen` 用于 30 分钟级别的早期项目初筛。它不替用户做投资决策，只把现有素材中的事实、缺口、初步判断和反向假设整理清楚。

## 输入

用户可提供以下一种或多种材料：

- deck、BP、官网文字、新闻稿、融资报道
- 会议逐字稿或 `/meeting-notes` 生成的访谈纪要
- 用户的初步印象、直觉、疑问或已知反面信息
- 公开赛道资料或可比公司信息

如果素材不足，不要补全空白；用 `[需要用户提供]` 或 `[未核实]` 标出来。

## 工作流

1. 先列出已读取素材，并区分 `00-source/` 原始材料、用户口述、公开信息、推测。
2. 提取一页事实摘要，所有数字和关键事实必须附来源标签。
3. 按七个维度初筛：
   - 市场
   - 痛点
   - 方案
   - 团队
   - 时机
   - 商业模式
   - 显性风险
4. 每个维度用同一结构输出：
   - 素材里说了什么
   - 缺什么
   - 初步判断
   - 如果这个判断错了，可能是什么样
5. 写出至少 2 条 bear case 雏形，避免只列优点。
6. 生成后续问题清单，问题要能直接拿去问创始人、客户或行业人士。
7. 单独列出 `[未核实]` 清单，方便用户补证据。

## 输出格式

使用 `templates/prescreen-template.md` 作为固定结构。

核心段落必须包含：

- `一页事实摘要`
- `七维初筛判断`
- `Bear Case 雏形`
- `未覆盖的关键问题`
- `[未核实] 清单`

## 质量红线

- 不写“建议投资”“不建议”“pass 掉”等终局判断。
- 不编造市场规模、估值、融资金额、用户数、增长率。
- 创始人未经验证的说法必须标 `[创始人自述]`。
- 用户表达的直觉要保留，但标为 `[用户直觉]` 或 `[推测]`，并追问具体触发点。
- 对项目或创始人的负面判断如果只来自 1–2 个信源，必须标 `[待交叉验证]`。
