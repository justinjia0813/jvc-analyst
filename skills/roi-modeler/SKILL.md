# /roi-modeler 投资回报模型

`/roi-modeler` 用于计算单笔 VC 投资在不同退出情形下的回报。它只建模，不替用户判断是否投资。

## 输入

- 本轮投资金额、投前/投后估值、目标持股比例
- 未来五年收入、毛利、EBITDA、现金消耗等财务预测
- 后续融资轮次、融资金额、估值、稀释假设
- 目标退出年份、退出估值方法、收入倍数或 EBITDA 倍数
- 可选：`/comps-dd` 可比公司估值倍数

缺失假设必须标 `[需要用户提供]` 或 `[未核实]`。

## 输出 Workbook

最终交付 `.xlsx`，按 `templates/roi-modeler-template.md` 的 schema 生成。

必须包含：

- `investment_terms`
- `financial_forecast`
- `financing_dilution`
- `exit_scenarios`
- `ownership`
- `returns`
- `sensitivity`
- `sources`

## 三情形要求

必须计算目标退出年的三种情形：

- **保守**：收入/利润低于计划、估值倍数收缩、稀释较高
- **中性**：接近基准预测
- **乐观**：收入/利润超预期、估值倍数较高、稀释可控

每个情形至少输出：

- 退出企业价值或股权价值
- 投资人退出时持股比例
- 投资人回款金额
- MOIC（Multiple on Invested Capital，投资回报倍数）
- IRR（Internal Rate of Return，内部收益率）

## 方法约束

- 融资稀释要逐轮计算，不只用最终持股拍脑袋。
- 每个后续融资轮必须注明融资额、估值、是否参与跟投、稀释比例。
- 退出估值倍数必须有来源或标 `[未核实]`。
- 保守/中性/乐观三情形的假设差异要写清楚，不能只改最终估值。
- 不输出“值得投”“不值得投”等结论，只输出回报区间和敏感项。

## 反馈迭代空间

真实项目跑完后，优先调整：

- 是否需要加入期权池扩张、清算优先权、跟投权、反稀释
- IRR 计算口径是否符合用户基金内部习惯
- 后续融资稀释假设是否过粗
- 输出是否能直接接入 `/ic-memo` 的估值与条款章节
