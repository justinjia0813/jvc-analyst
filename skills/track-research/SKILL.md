# /track-research 产业知识图谱

`/track-research` 用于快速建立一个细分赛道的认知框架。它回答“这个行业是什么、怎么发展到今天、技术路线如何分化、产业链怎么分工、趋势和风险在哪里”。

## 输入

- 赛道名称或细分问题，例如“工业视觉质检”“固态电池硫化物电解质”
- 可选：用户已知玩家、关注环节、地域范围、投资阶段
- 可选：已有公开报告、新闻、公司材料

默认范围：国内为主，海外标杆作为参照；公开资料优先，本地项目素材不外发。

## 输出

使用 `templates/track-research-template.md` 输出 Markdown。

必须覆盖：

- 行业定义与边界
- 行业简史
- 技术路线
- 产业链图谱
- 产业趋势
- 关键玩家
- 监管/政策/标准
- 投资相关问题
- 未核实与待补证据

## 方法约束

- 先定义边界，再写规模和玩家，避免把上游、下游、相邻赛道混算。
- 技术路线要按互斥分类写，不能把同一公司重复计入多个主路线，除非明确说明其多路线布局。
- 趋势要区分 `已经发生`、`正在发生`、`可能发生`。
- 所有市场规模、融资、收入、产能、价格、渗透率必须标来源。
- 找不到来源时标 `[未核实]`，不要补数字。
- 行业简史只保留影响产业结构的节点，不写百科式流水账。

## 反馈迭代空间

真实项目跑完后，优先调整：

- 技术路线分类是否足够正交
- 产业链图谱是否能直接指导 `/comps-dd`
- 关键玩家表是否缺少用户关心的新公司
- 趋势判断是否过于宏观，无法转化成投资问题
